# Optimize-WeakPC.ps1 — оптимизация слабого ПК на Windows 10/11

Самостоятельный PowerShell-скрипт, который **сам определяет железо** и **сам выбирает безопасные твики** под конкретную машину. Никакого ручного ввода: запустил — он сам всё проанализировал и применил.

> ⚠️ **Важно:** скрипт ещё не запускался на живой Windows (разрабатывался в Linux-окружении). Первый прогон делайте **только с `-DryRun`** и смотрите лог. Запускать **от имени администратора**.

## Что делает скрипт

1. **Определяет железо** (`Get-CimInstance` / `Get-PhysicalDisk`):
   - объём ОЗУ и модули памяти;
   - процессор (модель, ядра, потоки);
   - тип каждого диска — **HDD или SSD**;
   - видеокарту (дискретная или интегрированная);
   - версию и сборку Windows.
2. **Адаптивно подбирает твики** под найденное железо (см. [таблицу ниже](#какие-твики-применяются)).
3. **Применяет** их идемпотентно (повторный запуск ничего не ломает) и **ведёт лог**.

## Как запустить

### Режим `-DryRun` (пробный, безопасный) — делайте первым!
```powershell
powershell -ExecutionPolicy Bypass -File .\Optimize-WeakPC.ps1 -DryRun
```
Скрипт пройдёт весь путь, по каждому действию напишет `БЫЛО БЫ ИЗМЕНЕНО: ...`, но **ничего не тронет**: ни реестр, ни службы, ни питание, ни дефраг. Точка восстановления в этом режиме **не создаётся**.

### Боевой режим
```powershell
powershell -ExecutionPolicy Bypass -File .\Optimize-WeakPC.ps1
```
Порядок работы:
1. Проверка прав администратора.
2. Автодетект железа + вывод на экран.
3. **Создание точки восстановления** (если не удалось — стоп до любых изменений).
4. Применение адаптивных твиков.
5. Перезапуск `explorer.exe`, чтобы визуальные настройки вступили в силу.

После завершения рекомендуется **перезагрузка**.

## Какие твики применяются

Скрипт решает сам — на основе найденного железа:

| Условие | Что делает | Логика |
|---|---|---|
| **Всегда** | План питания → «Высокая производительность»; визуальные эффекты → «быстродействие» | база |
| **ОЗУ < 8 ГБ** | Отключает анимации; переводит в Manual/Disabled безопасные фоновые службы (телеметрия, факс, демо-режим и т.п.) | агрессивнее экономит RAM |
| **ОЗУ < 4 ГБ** | Дополнительно отключает индексатор поиска `WSearch` | очень мало памяти |
| **Есть HDD** | Включает штатную задачу оптимизации дисков по расписанию (дефрагментация) | HDD выигрывает от дефрага |
| **Только SSD** | Дефраг **НЕ трогает** (вредно для SSD), проверяет/включает TRIM | логика противоположна HDD |
| **Слабый/интегрированный GPU** | Отключает прозрачность и анимации интерфейса | разгружает видеоядро |

> Тип диска определяется по `MediaType`, а при неопределённости — по `SpindleSpeed`. Если тип установить не удалось — дисковые твики **пропускаются** (безопасно). USB/SD-носители в расчёт не берутся.

## Безопасность

Что скрипт **гарантированно НЕ трогает** (защищённый «чёрный список» служб):
- **Windows Update** (`wuauserv`, `bits`, `UsoSvc`, `DoSvc`);
- **Microsoft Defender** и безопасность (`WinDefend`, `SecurityHealthService`, `Sense`, `WdNisSvc`, `wscsvc`);
- **Брандмауэр** (`mpssvc`);
- **Microsoft Store** (`InstallService`, `AppXSvc`, `ClipSVC`).

Другие принципы:
- **Точка восстановления** создаётся до любых изменений. Не удалось — скрипт останавливается.
- **Идемпотентность**: если значение уже выставлено, оно пропускается. Повторный запуск безопасен.
- **Каждая правка реестра** логируется с исходным значением (`# откат: вернуть '...'`).
- Временно снятый лимит частоты точек восстановления **возвращается** в исходное состояние.

## Лог и откат изменений

**Лог** каждого запуска: `C:\OptimizeLog\optimize_ГГГГ-ММ-ДД_ЧЧММСС.log`
В нём для каждого действия видно «было → стало» и подсказка по откату.

**Как откатить:**
1. **Самый простой способ** — откатиться на созданную точку восстановления:
   `Пуск → "Создание точки восстановления" → Восстановить...`
2. **Точечно** — по логу: там для каждой правки указано исходное значение и команда возврата (например, `powercfg /setactive <старый-GUID>` для плана питания).

## Частые вопросы

**Скрипт не запускается, ошибка про политику выполнения.**
Используйте флаг `-ExecutionPolicy Bypass`, как в примерах. Он действует только на этот запуск и ничего не меняет в системе.

**Кириллица в выводе — «кракозябры».**
Файл сохранён в UTF-8 с BOM и принудительно включает UTF-8 в консоли — на обычной Windows всё читается. Если консоль совсем старая, помогает `chcp 65001` перед запуском.

**Можно ли запускать на ноутбуке?**
Да, но план питания «Высокая производительность» расходует батарею быстрее. Исходную схему можно вернуть по логу.

**Безопасно ли это?**
Скрипт не трогает Update/Defender/Store/брандмауэр, делает точку восстановления и ведёт лог. Но он непроверен на живой Windows — **первый прогон делайте с `-DryRun`**.
